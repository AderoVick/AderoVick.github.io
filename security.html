Security

Secrets stay on the server.

The platform is designed so public pages contain no payment secret keys, social access tokens or database service credentials.

Production checklist

  • Deploy dynamic APIs through Cloudflare Pages Functions or Workers.
  • Store secrets with Cloudflare encrypted environment variables.
  • Use Supabase Row Level Security and a private file bucket.
  • Verify Stripe, PayPal and M-Pesa callbacks before marking payments paid.
  • Restrict social publishing endpoints to an authenticated admin role.
  • Enable multi-factor authentication on GitHub, Cloudflare, Supabase and payment accounts.
  • Review logs and rotate exposed or unused credentials immediately.

Never submit

  • Passwords, card PINs or mobile-money PINs.
  • API secret keys or access tokens.
  • Crypto recovery phrases or private keys.
  • Unredacted identity documents unless specifically required and securely arranged.
  • Data you are not authorised to share.
Report a suspected vulnerability privately by email. Do not include active credentials or private client data in the report.
